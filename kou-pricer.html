01 · Model Overview

Why do we need
jump-diffusion?

Black-Scholes assumes stock prices move on a continuous path — no sudden teleportation. Real markets do jump: earnings surprises, central bank shocks, geopolitical events. Kou (2002) introduced a tractable jump-diffusion model whose jump sizes follow a double-exponential distribution, yielding closed-form option prices.

BLACK-SCHOLES BASELINE

Continuous diffusion only

The classic model assumes dS = μS dt + σS dW. Paths are continuous — no jumps. This underprices out-of-the-money options (the "volatility smile") because it ignores fat tails and sudden moves.

KOU (2002) EXTENSION

Diffusion + Poisson jumps

Kou adds a compound Poisson process: jumps arrive at rate λ per year. Each jump multiplies S by a random factor J drawn from a double-exponential distribution — asymmetric, with separate rates for up and down jumps.

KEY INNOVATION

Double-exponential jump sizes

Unlike Merton's lognormal jumps, the double-exponential gives a leptokurtic distribution (sharp peak, heavy tails) and remains analytically tractable. The memoryless property of the exponential lets us compute first-passage times in closed form.

PRACTICAL RESULT

Fits the volatility smile

The asymmetry parameter p and the decay rates η₁, η₂ let the model match both the skew (more down-jumps than up) and the kurtosis (how fat the tails are) observed in real equity markets.

// Kou SDE under risk-neutral measure
dS/S = (r − q − λκ) dt + σ dW + d(Σ jumps)

// Jump size distribution — log(J) ~ double-exponential
f(y) = p · η₁ · exp(−η₁ y) if y > 0 (upward jump)
f(y) = (1−p) · η₂ · exp(η₂ y) if y < 0 (downward jump)

// Martingale correction (ensures E[J] = 1+κ)
κ = p·η₁/(η₁−1) + (1−p)·η₂/(η₂+1) − 1
02 · Algorithm Walkthrough

How the simulation
is built, step by step

01

Set up the risk-neutral drift

Under the risk-neutral measure, the expected return must equal the risk-free rate r. The jump component introduces an extra expected return of λκ per unit time. We subtract this from the drift so the discounted stock price remains a martingale:

μ_RN = r − q − λ · κ
// q = continuous dividend yield, κ = mean jump size compensation

Without this correction the simulated stock would systematically drift upward (or downward), producing biased prices. Every time step we apply this corrected drift.

02

Simulate the Brownian diffusion component

Over a small time step dt = T/N, the continuous diffusion part contributes:

ΔlogS_diffusion = (μ_RN − ½σ²) dt + σ · √dt · Z
// Z ~ N(0,1), generated by Box-Muller or inverse-CDF method

We work in log-space (log returns) because the lognormal is the natural state space for multiplicative price changes. Euler discretisation in log-space is exact for the diffusion part — no discretisation error here, only from the jump approximation.

03

Simulate Poisson-distributed jump arrivals

The number of jumps in [t, t+dt] follows a Poisson distribution with mean λ·dt. We draw from this distribution for each time step:

N_jumps ~ Poisson(λ · dt)
// For small λ·dt, usually 0 jumps. Occasionally 1 or 2.
// Draw: while product of U[0,1] > e^(-λdt), increment count

For λ = 1 and daily steps dt ≈ 1/252, the expected number of jumps per step is only 0.004 — so most steps have zero jumps. This explains why paths look mostly smooth with occasional vertical spikes.

04

Draw double-exponential jump sizes

For each jump that arrives, we draw its log-size from the double-exponential distribution:

With probability p: Y ~ Exp(η₁) (positive — upward jump)
With probability 1-p: Y ~ −Exp(η₂) (negative — downward jump)

// Inverse CDF sampling: Y = −log(U) / η where U ~ Uniform(0,1)
// Mean up-jump: +1/η₁ (e.g., η₁=10 → mean +10%)
// Mean down-jump: −1/η₂ (e.g., η₂=5 → mean −20%)

The total log-return for one time step is: Δlog S = diffusion part + Σ Y_i over all jumps that arrived. We then set S_{t+dt} = S_t · exp(Δlog S).

05

Compute the exotic payoff at expiry

Once a full path S₀, S₁, …, S_N is simulated, we evaluate the option-specific payoff:

// European call: max(S_T − K, 0)
// European put: max(K − S_T, 0)
// Down-and-out barrier call: max(S_T − K, 0) · 1{min S > H}
// Digital cash-or-nothing call: 1{S_T > K}
// Asian arithmetic call: max( (1/N)Σ S_i − K, 0 )

For barriers under jump-diffusion, we check whether the path breached H at any monitored step. Note: if the stock jumps over the barrier without the price being at H at a monitored time, it counts as a breach only if monitoring is daily. This is the "barrier discretisation" effect unique to jump models.

06

Discount and average across paths

The Monte Carlo estimate is simply the average discounted payoff:

Price_MC = e^(-rT) · (1/M) · Σᵢ payoff_i

// Standard error: SE = e^(-rT) · std(payoffs) / √M
// 95% CI: [Price − 1.96·SE, Price + 1.96·SE]

With M = 20,000 paths and typical parameters, the standard error is around 0.01–0.05 for at-the-money options. Increasing to 50,000 paths halves the error (error scales as 1/√M).

07

Semi-analytical pricing for Europeans (Kou's formula)

For European options only, Kou derived a semi-closed-form: we expand the price as an infinite sum over the number of jumps that occurred, each weighted by its Poisson probability:

Price_Kou = Σ_{n=0}^{∞} w_n · BS(S, K, T, r_n, q, σ_n)

// w_n = exp(−λ*T) · (λ*T)^n / n! — Poisson weights (λ* = λ(1+κ))
// σ_n² = σ² + n·Var[Y]/T — augmented variance with n jumps
// r_n = r − λκ + n·log(1+κ)/T — adjusted drift for n jumps

The series converges quickly — typically 20–30 terms suffice. This gives a fast benchmark to validate the Monte Carlo. If MC and analytical disagree by more than 2×SE, check your path count or step size.

04 · Deep Dives

Concepts explained

What is the martingale correction κ?

+

In risk-neutral pricing, the discounted asset price must be a martingale — its expected future value (discounted) equals its current value. The continuous diffusion part is already a martingale if we set the drift to r − q − ½σ². But adding jumps with mean size κ introduces extra expected growth of λκ per unit time. We subtract this from the drift to compensate:

κ = E[J − 1] = p·η₁/(η₁−1) + (1−p)·η₂/(η₂+1) − 1
// This requires η₁ > 1 (otherwise mean up-jump is infinite!)

If you set λ = 0, κ drops out entirely and you recover Black-Scholes drift. The correction only matters when jumps are large (small η₁ or η₂) or frequent (large λ).

Why do barriers behave differently under jumps?

+

Under continuous diffusion, the stock must pass through the barrier level to breach it — so continuous and daily-monitored barriers give similar prices. Under jump-diffusion, the stock can teleport past the barrier in a single jump without ever touching it. Two effects:

  • More knock-outs: downward jumps can breach a down-and-out barrier more easily, reducing the price of knock-out options.
  • Jump-over effect: with daily monitoring, a jump that crosses H and comes back won't be detected. This makes knock-out barriers less sensitive to monitoring frequency than in pure diffusion models.

Try setting η₂ = 3 (mean −33% downward jump) and a barrier at 90. You'll see barrier prices collapse because almost any downward jump knocks the option out immediately.

How does the volatility smile arise from the Kou model?

+

The Black-Scholes implied volatility (IV) "should" be flat if BS were true. In reality, out-of-the-money puts trade at higher IV than ATM options (the "skew"), and OTM calls also have elevated IV (making a "smile"). Kou explains this:

  • OTM puts: asymmetric down-jumps (p < 0.5, large η₂) add left-tail probability → higher OTM put prices → higher implied vol for low strikes.
  • Short maturities: at 1-week maturity, jumps dominate (not enough time for diffusion to smooth them) → extreme smile.
  • Long maturities: the Central Limit Theorem takes over — many small jumps look like extra diffusion → smile flattens.

You can observe this by pricing European puts at different strikes (changing K) and computing what BS σ would give the same price — that's the implied vol.

Why are Greeks numerically computed here, not analytically?

+

For European options, Kou's analytical formula can be differentiated to get analytical Greeks. But for barrier, digital, and Asian options there is no closed-form derivative. We use finite differences:

Delta ≈ [V(S+h) − V(S−h)] / (2h) // central difference, h = 0.5
Gamma ≈ [V(S+h) − 2V(S) + V(S−h)] / h²
Theta ≈ [V(T−1/365) − V(T)] / (1/365) // forward difference

Each Greek requires 2–3 additional MC runs (using 5,000 paths for speed). The path count for Greek estimation doesn't need to be as large as the price estimate — but watch for noise when Gamma is near zero or at barriers.

How would you calibrate the model to market data?

+

Calibration means finding the parameter vector (σ, λ, p, η₁, η₂) that minimises the squared difference between model prices and market prices across a set of liquid options:

min_{σ,λ,p,η₁,η₂} Σᵢ wᵢ [V_model(Kᵢ, Tᵢ; θ) − V_market(Kᵢ, Tᵢ)]²
// wᵢ = 1/bid-ask spread² (weight by liquidity)
subject to: σ > 0, λ ≥ 0, 0 < p < 1, η₁ > 1, η₂ > 0

Typical approach: use the analytical European formula (fast) to calibrate on vanilla options, then price exotics by MC using the calibrated parameters. Differential evolution or Levenberg-Marquardt are common optimisers. The constraint η₁ > 1 is critical — otherwise the mean jump is infinite.
